# 터미널 없이, 클로드 앱만으로 에이전트 만드는 법

> Claude 데스크탑 앱(Co-work)으로 터미널 없이 지능형 에이전트 시스템을 구축하는 폴더 아키텍처·파일 세팅 가이드

---

## 🚀 클로드 활용의 3단계

| 단계 | 명칭 | 역할 | 특징 |
|------|------|------|------|
| **Level 1** | **Chat (채팅)** | 단순 챗봇 | 사용자가 계속 프롬프트를 넣어야 함 (내가 엔진이 되는 단계) |
| **Level 2** | **Co-work (코워크)** | **개인 비서** | 내 컴퓨터의 로컬 폴더를 읽고, 스케줄링(예약 작업)이 가능함 |
| **Level 3** | **Claude Code** | **엔지니어** | 코드를 직접 짜고, 도구/웹사이트/자동화 워크플로우를 직접 구축함 |

### Level 2: Co-work (나의 유능한 비서)

- **기능:** 내 컴퓨터의 특정 폴더에 접근해 파일을 정리, 요약, 분석
- **Scheduled Tasks:** 매일 아침 이메일 요약, 매달 가계부 정리 등 반복 작업 예약 (컴퓨터가 켜져 있어야 작동)
- **모바일 활용:** 'Dispatch' 기능으로 스마트폰에서 내 컴퓨터(Co-work)에 명령 가능

### Level 3: Claude Code (나의 전담 개발자)

- **기능:** 단순 파일 읽기를 넘어, 새로운 프로그램·대시보드를 직접 생성
- **예시:** 지출 내역 파일 → 자동 시각화 재무 대시보드, 경쟁사 채널 분석 → 콘텐츠 기획안 도구

---

## 핵심 시스템 구성 — 폴더 아키텍처 세팅

프롬프트 엔지니어링보다 **파일 세팅(Context Engineering)** 이 더 중요합니다.
모든 프로젝트는 개별 폴더를 가지며, Claude는 아래 3가지 파일로 문맥을 파악합니다.

- **`INSTRUCTIONS.md`** (직원 핸드북): Claude의 역할, 업무 범위, 응답 규칙, 선호 출력 형식 정의
- **`MEMORY.md`** (장기 기억): 세션을 거듭하며 배운 취향·수정사항·결정사항을 저장하는 '뇌'
- **`CONTEXT-strategy.md`** (참조 문서): 비즈니스 전략, 브랜드 보이스, 과거 스크립트 등 참고 자료
- **`/outputs/`**: 결과물이 저장되는 폴더

### CLAUDE.md vs INSTRUCTIONS.md 비교

|  | `CLAUDE.md` | `INSTRUCTIONS.md` |
|--|------------|------------------|
| 어디서 쓰나 | **Claude Code** (터미널) | **Co-work** (데스크탑 앱) |
| 누가 자동으로 읽나 | Claude Code가 세션 시작 시 자동 인식 | Co-work에서 폴더 연결 시 자동 인식 |
| 위치 | `~/.claude/CLAUDE.md` 또는 프로젝트 루트 | 프로젝트 폴더 안 어디든 |
| 함께 쓰는 파일 | 없음 (단독으로도 충분) | `MEMORY.md` + `CONTEXT.md`와 세트 |

**정리:**
- 개발 저장소 → `CLAUDE.md`
- 비개발 업무 폴더 → `INSTRUCTIONS.md`
- 장기 취향/교정 기록 → `MEMORY.md`
- 브랜드·업무 배경 문서 → `CONTEXT.md` / `STRATEGY.md`

한 프로젝트 안에서는 **한 가지 체계로 통일**하는 게 핵심입니다.

## 핵심 파일 템플릿

### [INSTRUCTIONS.md] — 범용 지침 템플릿

```
# Role & Identity
You are my [분야] Agent.
Your goal is to help me [수행할 핵심 목표 1-2가지].

# Core Mission
- [수행할 작업 1]
- [수행할 작업 2]
- [수행할 작업 3]

# Response Rules
1. ALWAYS match my tone: [선호하는 말투]
2. Be concise: Avoid AI-sounding filler and keep responses focused on utility.
3. Quality Control: Double-check logic and technical accuracy before providing results.
4. Language: Use [선호 언어]

# What Good Output Looks Like
- [포맷/길이/스타일 선호도]

# The Absolute Memory Rule (CRITICAL)
At the start of every session, read MEMORY.md to understand my current preferences
and past corrections. At the end of every session, when I ask you to update MEMORY.md,
record any new preferences, corrected mistakes, or established patterns from our interaction.
```

### [MEMORY.md] — 범용 기억 템플릿

```
# MEMORY - Self-Updating Knowledge Base

## 1. User Preferences
- [클로드가 파악한 사용자의 선호도가 여기에 기록됩니다]

## 2. Corrections & Lessons Learned
- [클로드가 실수한 내용과 사용자가 교정해준 정답이 기록됩니다]

## 3. Established Patterns
- [반복적으로 요청하는 작업의 패턴이 기록됩니다]

## 4. Key Decisions
- [함께 내린 중요한 결정이나 확정된 계획이 기록됩니다]
```

**작동 방식:** 세션이 끝날 때 "MEMORY.md 업데이트해줘"라고 요청하면 Claude가 그날 파악한 내용을 파일에 직접 기록합니다.

### [CONTEXT-strategy.md] — 범용 템플릿

```
# Context Strategy

## Business/Project Overview
- [비즈니스 또는 프로젝트의 핵심 목표]
- [현재 가장 집중하고 있는 우선순위]

## Brand & Voice
- [선호하는 말투와 커뮤니케이션 스타일]
- [반드시 지켜야 할 가이드라인이나 피해야 할 표현]

## Operations & SOPs
- [업무 처리를 위한 표준 단계나 절차]
- [주로 사용하는 도구 및 소프트웨어 목록]

## Past Data & References
- [참고할 만한 성공적인 과거 작업물 목록]
- [벤치마킹 중인 사례나 경쟁사 모델]
```

**컨텍스트 파일에 넣을 내용 예시:**
- 콘텐츠 크리에이터: 콘텐츠 전략, 브랜드 보이스, 과거 스크립트
- 비즈니스 소유자: 회사 개요, SOP, 제품 문서
- 프리랜서: 클라이언트 브리프, 포트폴리오 요약, 요율표
- 개발자: 아키텍처 문서, 코딩 표준, API 명세서

> 핵심 질문: "신입 사원에게 첫날 건네줄 자료가 무엇인가?"

## 🔄 프로의 루프 (The Pro Loop)

Co-work와 Claude Code를 결합해 수동 업무를 줄여나가는 과정:

1. **실무 수행:** Co-work에 실제 업무(가계부 분석, 대본 초안 등)를 맡깁니다.
2. **패턴 포착:** 반복적으로 같은 종류의 일을 시키고 있다는 것을 인지합니다.
3. **자동화 전환:** 그 신호가 오면 Claude Code로 이동합니다.
4. **시스템 구축:** Claude Code로 해당 업무를 영구적으로 자동화하는 시스템을 만듭니다.
5. **역할 변화:** 이제 Co-work는 자동화된 결과물을 검토(Review)하는 역할로 바뀝니다.
6. **반복:** 이 루프를 반복할수록 수동 작업은 줄어듭니다.

### 복리 효과 타임라인

- **1주차:** 개별 작업들을 하나씩 에이전트에게 넘기기 시작
- **4주차:** 특정 업무 카테고리 전체를 처리하는 시스템 구축
- **8주차:** 컴퓨터가 켜져 있는 동안 에이전트들이 자동으로 성과 축적

## ✅ 20분 퀵스타트 체크리스트

- [ ] **프로젝트 폴더 생성:** 전용 폴더를 만들고, 3개 파일(`INSTRUCTIONS.md`, `MEMORY.md`, `CONTEXT-strategy.md`) 생성
- [ ] **INSTRUCTIONS.md 작성:** 템플릿 참고해 Claude의 역할과 규칙 정의
- [ ] **MEMORY.md 생성:** 선호도/교정/패턴/결정 4개 섹션 생성
- [ ] **CONTEXT-strategy.md 추가:** 내 업무와 관련된 문서 삽입
- [ ] **Co-work 실행:** 오늘 해야 할 진짜 업무 하나를 던져줍니다
- [ ] **패턴 찾기:** 반복 작업이 생기면 Claude Code로 넘어가 자동화 도구로 전환
