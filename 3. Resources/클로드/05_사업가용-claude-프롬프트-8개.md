# 사업가용 Claude 프롬프트 8개 — 매출 2배 전략 세트

> 비즈니스 컨설팅 수준의 Claude 프롬프트 8개로 주말 반나절에 분기 전략 전체를 수립하는 실전 세트

---

이 프롬프트들은 실제 비즈니스 컨설팅 수준의 깊이로 돌아갑니다. 주말 반나절 잡고 순서대로 돌리면, 내 사업의 2026년 2분기 전체 전략이 1시간 안에 나옵니다.

---

## 💰 PROMPT 1 · 매출 레버 스캐너

```
내 비즈니스 모델을 분석해서 매출 2배 만들 수 있는
최고 임팩트 레버를 찾아줘.

아래 5개 축으로 나눠서 분석:
1. 트래픽 확보 기회 (신규 고객 유입)
2. 전환율 개선 (방문→구매)
3. 가격 최적화 (객단가·패키지)
4. 신규 오퍼·번들 확장
5. 리텐션·LTV

각 레버별 매출 임팩트 추정(%) + 6~12개월 내
실현 가능성 평가. TOP 3 레버만 뽑고
각각 구체적 실행 플랜 만들어줘.

내 정보:
- 비즈니스 모델: [B2B SaaS / D2C / 교육 / 컨설팅 등]
- 현재 월 매출: $[금액]
- 오디언스 규모: [숫자]
- 주력 상품 1개 설명: [가격·마진·판매량]
- 현재 제일 큰 병목: [설명]
```

**사용 팁**: 매출 단위는 원으로 바꿔도 OK. 정확한 숫자 안 알려줘도 대략적 범위로도 결과 나와요.

## 🎯 PROMPT 2 · 시장 지배 플랜 (18개월 로드맵)

```
내 카테고리에서 1위가 되기 위한 18개월 로드맵 만들어줘.

아래 5단계로 나눠서:

1. 현재 포지션 진단 (지금 내가 시장 몇 등인가)
2. 진입점 전략 (가장 얇은 경쟁 지점)
3. 초기 6개월 목표 + 핵심 지표
4. 중기 6~12개월 확장 (인접 시장·파트너십)
5. 12~18개월 카테고리 리더십

각 단계별:
- 핵심 활동 3개
- 투자 우선순위
- 측정 지표 (OKR 형식)
- 위험 요소 + 대응

내 카테고리: [예: 중소기업 AI 도입 컨설팅]
경쟁사 TOP 3: [이름]
차별화 포인트: [설명]
```

## 🕵 PROMPT 3 · 경쟁사 디코더

```
아래 경쟁사 3곳 해부해줘.
(각 경쟁사 홈페이지 URL·SNS·대표 상품 링크)

경쟁사 1: [URL]
경쟁사 2: [URL]
경쟁사 3: [URL]

아래 6가지 기준으로 분석:

1. 오퍼 구조 (상품 라인·가격 티어·번들)
2. 핵심 메시지 (3문장으로 요약)
3. 포지셔닝 (누구를 타겟, 무엇을 약속)
4. 강점 3가지 / 약점 3가지
5. 가격 전략 (진입가·확장가·프리미엄)
6. 마케팅 채널 (SEO·광고·SNS·PR)

마지막에:
- 공통 약점 TOP 5 (내가 파고들 빈틈)
- 차별화 포지션 3가지 제안
- 카피 문장 예시 3개 (내 메시지로 변환)
```

**사용 팁**: Firecrawl MCP 있으면 URL만 주면 자동 크롤링. 없으면 홈페이지 텍스트 직접 복붙.

## 🔥 PROMPT 4 · 바이럴 시스템 설계

```
내 상품에 맞는 바이럴 구조 설계해줘.

아래 3가지 바이럴 메커니즘 중 내 상품에 맞는
3가지 조합 추천 + 구현 방법:

1. 리퍼럴 (초대 시스템)
2. PLG (Product-Led Growth — 제품 자체가 전파)
3. 쉐어 트리거 (결과물 공유 유도)
4. 네트워크 효과 (많은 사용자 = 가치 증가)
5. 커뮤니티 빌딩

각 메커니즘:
- 내 상품에 맞는 이유
- 구현 순서 (1~4주차)
- 필요 도구·예산
- 측정 지표 (K-factor, Viral Coefficient 등)
- 1개월 내 실행 가능한 MVP

내 상품: [설명]
현재 고객 수: [숫자]
평균 NPS: [점수 또는 모름]
```

## 💎 PROMPT 5 · 고전환 오퍼 설계

```
내 상품 5개를 "프리미엄 래더" 구조로 재설계해줘.

목표: 각 가격 티어에서 자연스럽게 위로 올라가게.

구조:
- 엔트리 ($99 이하): 진입 장벽 낮추는 리드 마그넷
- 코어 ($99~$499): 주력 상품
- 프리미엄 ($500~$2,000): 고수익 패키지
- VIP ($2,000+): 1:1·커뮤니티·장기

각 티어별:
- 상품 구성 (3~5가지 구성 요소)
- 가격 정당화 카피 (2문장)
- 번들 보너스 (3가지)
- 구매 저항 제거 (보장·환불·사례)
- 업셀 시나리오 (다음 티어로 유도)

내 현재 상품 5개:
1. [이름 / 가격 / 핵심 가치]
2. ...
```

## 🤖 PROMPT 6 · 자동화 전략가

```
내 일주일 업무를 분석해서 자동화 가능한 것 뽑고
실행 플로우 설계해줘.

아래 내 하루 업무 리스트에서:
1. 자동화 가능 업무 10개 이상 선별
2. 각 업무별 추천 툴 조합 (Zapier / Make / Claude / n8n / Slack 등)
3. 구현 난이도 (1~5)
4. 월 절약 시간 추정
5. 월 운영 비용 추정
6. ROI 높은 순 TOP 5만 우선 실행 로드맵

각 TOP 5 업무:
- 현재 수동 플로우
- 자동화 후 플로우
- 단계별 구현 (Trigger → Action → Output)
- 체크포인트 (중간 검수 포인트)

내 하루 업무:
1. [오전 9시: ...]
2. ...
(최대한 구체적으로)
```

## 🧠 PROMPT 7 · 고객 페인 인텔리전스

```
내 고객 데이터에서 진짜 "돈 쓰는 이유" 5개 뽑아줘.

데이터:
- 구매 후 리뷰 20개
- 환불 사유 10개
- 판매 전 DM·문의 20개
- NPS 댓글
- 경쟁사 부정 리뷰

[데이터 복붙]

분석:
1. 감정 단어 빈도 TOP 30
2. 진짜 통증 TOP 5 (표면적 vs 심층적 분리)
3. 구매 결정 트리거 (무엇이 지갑을 열게 하는가)
4. 이탈·환불 공통 원인
5. 각 페인에 대응하는 카피 문장 3개씩 (광고·랜딩·이메일용)

마지막:
- 내 상품 설명 페이지 재작성 (페인 기반)
- 광고 카피 5개 (각 페인 1개씩)
```

## 📈 PROMPT 8 · 프로핏 엔진

```
매출 100만 원당 실제 남는 돈 분석해서
수익률 2배 만드는 최적화 플랜 뽑아줘.

내 비용 구조 (월간):
- 매출: $[금액]
- 원가 / COGS: $[금액]
- 마케팅·광고: $[금액]
- 인건비: $[금액]
- 플랫폼·구독: $[금액]
- 기타 운영비: $[금액]
- 실제 순이익: $[금액] ([%])

분석 지표:
1. Gross Margin
2. CAC (Customer Acquisition Cost)
3. LTV (Lifetime Value)
4. LTV:CAC 비율
5. Payback Period

최적화 레버 3개 우선순위 선정:
- 가격 인상 가능 여부 (수요 탄력성 추정)
- 원가 절감 포인트
- 광고 ROAS 개선 (채널별 분석)
- 리텐션·리피트율 증가

각 레버별:
- 실행 플랜 (30일 / 60일 / 90일)
- 기대 수익률 개선 (%)
- 리스크 요소
```

## 🗓 주말 반나절 실행 플랜

**토요일 오전 (2시간)**
1. PROMPT 1 (매출 레버 스캐너) → TOP 3 레버 정의
2. PROMPT 7 (고객 페인) → 카피 문장 확보

**토요일 오후 (2시간)**
1. PROMPT 3 (경쟁사 디코더) → 빈틈 3개 찾기
2. PROMPT 5 (고전환 오퍼) → 프리미엄 래더 초안

**일요일 오전 (2시간)**
1. PROMPT 2 (시장 지배 플랜) → 18개월 로드맵
2. PROMPT 4 (바이럴 시스템) → 3개월 실행 MVP

**일요일 오후 (2시간)**
1. PROMPT 6 (자동화 전략가) → TOP 5 자동화 로드맵
2. PROMPT 8 (프로핏 엔진) → 수익률 최적화 플랜

→ 월요일 아침 = 분기 전체 전략 실행 시작 준비 완료

## 💡 입력 템플릿 (모든 프롬프트 공통으로 위에 붙이기)

```
## 내 비즈니스 정보 (모든 프롬프트 공통)
- 사업 분야: [구체적으로]
- 창업 몇 년차:
- 월 평균 매출:
- 팀 규모:
- 주력 상품 1문장 소개:
- 타겟 고객 페르소나:
- 현재 가장 큰 고민 1가지:
```

이 블록을 매 프롬프트 위에 복붙하면 결과 품질이 3배 좋아집니다.

---

## 📌 주의사항

1. **개인정보 조심**: 고객 리뷰·이메일 등 붙일 때 개인정보 제거
2. **재무 민감 정보**: 상세 숫자는 비율·범위로 바꿔서 입력
3. **답변 검증**: Claude는 내 시장 실시간 데이터 없으므로 추정치. 실행 전 반드시 실제 데이터로 검증
4. **혼자 돌리기 vs 팀**: 팀 있으면 프롬프트별로 담당자 분배하고 합치세요
