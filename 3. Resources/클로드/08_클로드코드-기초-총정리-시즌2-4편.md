## 클로드코드 Hooks 개념 익히기

클로드 코드의 **Hooks**는 AI가 특정 행동을 할 때 자동으로 실행되는 규칙입니다. 공장의 자동 검수 라인과 같습니다. 직원(AI)이 제품(코드)을 만들면, 컨베이어 벨트가 알아서 품질 검사를 통과시키는 구조입니다.

Hooks가 없으면 AI가 코드를 수정한 뒤 포맷도 안 맞추고, 테스트도 안 돌리고 끝냅니다. 시킨 것만 하고 뒷정리를 안 하는 것입니다. 그래서 매번 사람이 직접 포맷 정리하고, 테스트 실행하고, 결과 확인까지 해야 했습니다.

Hooks를 설정하면 이 과정이 전부 자동화됩니다.

### 핵심 4가지 타이밍

| 타이밍 | 동작 시점 | 활용 예시 |
|--------|----------|----------|
| **PreToolUse** | 도구 실행 직전 | `rm -rf` 같은 위험한 명령어 자동 차단 |
| **PostToolUse** | 파일 저장 직후 | Prettier 같은 포맷터 자동 실행 |
| **Notification** | 작업 완료 시 | Slack 알림 발송 |
| **Stop** | 세션 종료 시 | 로그 정리 |

### 설정 방법

`.claude/settings.json`에 Hook 종류, 매칭 조건, 실행할 명령어를 작성하면 됩니다.

예를 들어 파일 쓰기가 발생할 때마다 자동으로 `npx prettier --write`를 실행하도록 한 줄만 넣어두면, 이후 모든 세션에서 코드 포맷이 자동으로 정리됩니다.

다른 AI 코딩 도구에는 없는 클로드 코드만의 차별화 기능이며, 매번 수동으로 검사하던 5분이 자동으로 0초가 됩니다.
