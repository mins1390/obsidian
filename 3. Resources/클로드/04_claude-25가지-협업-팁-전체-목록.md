# Claude 25가지 협업 팁 전체 목록

> CLAUDE.md 세팅부터 MCP 연동까지, 카테고리별로 정리한 Claude 실전 협업 팁 25가지

---

## 카테고리 1: 프로젝트 세팅 (1~5)

**1. CLAUDE.md로 프로젝트 컨텍스트 고정**

```markdown
# 프로젝트: [이름]
- 스택: [기술 스택]
- 컨벤션: [코딩 규칙]
- 폴더 구조: [구조 설명]
```

→ 매 대화마다 설명 반복 불필요

**2. /init으로 프로젝트 자동 분석**

```
/init
```

→ Claude가 프로젝트 구조를 자동으로 파악하고 CLAUDE.md 초안 생성

**3. 글로벌 설정으로 모든 프로젝트에 규칙 적용**

```
~/.claude/CLAUDE.md
```

→ 개인 코딩 스타일, 선호 패턴을 모든 프로젝트에 자동 적용

**4. .claude/rules/ 디렉토리로 규칙 계층화**

```
.claude/rules/
├── common/coding-style.md
├── typescript/patterns.md
└── web/performance.md
```

→ 공통 규칙 + 언어별 규칙을 체계적으로 관리

**5. 프로젝트별 설정 분리**

```
~/.claude/projects/{project-path}/settings.json
```

→ 프로젝트마다 다른 모델, 도구 권한 설정

---

## 카테고리 2: 일상 작업 효율화 (6~10)

**6. /compact로 컨텍스트 압축**

```
/compact
```

→ 긴 대화 중간에 토큰 절약

**7. Tab 자동완성 활용**

파일명, 명령어 입력 중 Tab 키 → 파일 경로, 명령어를 자동으로 완성

**8. ! 접두사로 쉘 명령어 직접 실행**

```
! npm run dev
```

→ Claude 대화 중에 터미널 명령어 바로 실행

**9. Shift+Tab으로 모델 빠르게 변경**

→ 간단한 작업은 Haiku, 복잡한 작업은 Opus로 즉시 전환

**10. /cost로 실시간 비용 확인**

```
/cost
```

→ 현재 세션의 토큰 사용량과 비용 즉시 확인

---

## 카테고리 3: 코딩 생산성 (11~15)

**11. 멀티 에이전트 병렬 실행**

```
메인: 기능 구현
에이전트 1: 테스트 작성
에이전트 2: 코드 리뷰
```

→ 3가지 작업을 동시에 처리

**12. Git 워크플로우 자동화**

```
/commit
```

→ 변경사항 분석 → 커밋 메시지 자동 생성 → 커밋

**13. 코드 리뷰 에이전트 활용**

```
"이 PR 리뷰해줘" + git diff
```

→ 보안, 성능, 코드 품질 자동 점검

**14. TDD 가이드 에이전트**

```
"이 기능을 TDD로 구현해줘"
```

→ 테스트 먼저 작성 → 구현 → 리팩토링 자동 진행

**15. 에러 디버깅 자동화**

에러 메시지 복붙만 하면 → 원인 분석 → 수정 → 테스트까지

---

## 카테고리 4: 고급 활용 (16~20)

**16. MCP 서버 연동**

```json
{
  "mcpServers": {
    "notion": { ... },
    "github": { ... }
  }
}
```

→ Notion, GitHub, Slack 등 외부 서비스 직접 연동

**17. 커스텀 슬래시 명령어 만들기**

```
~/.claude/skills/{skill-name}/SKILL.md
```

→ 자주 쓰는 워크플로우를 /명령어로 등록

**18. Hooks로 자동화 파이프라인**

```json
{
  "PostToolUse": [{
    "matcher": "Write|Edit",
    "command": "npx prettier --write $FILE_PATH"
  }]
}
```

→ 코드 저장할 때마다 자동 포맷팅

**19. Extended Thinking 활용**

`Option+T (macOS)` → 복잡한 문제에 깊은 추론 모드 활성화

**20. Plan Mode로 설계 먼저**

```
/plan
```

→ 코딩 전에 아키텍처와 구현 계획 수립

---

## 카테고리 5: 프로 팁 (21~25)

**21. 이미지/스크린샷으로 지시하기**

```
"이 디자인대로 만들어줘" + 스크린샷 첨부
```

→ Figma 시안을 바로 코드로 변환

**22. Git Worktree로 병렬 브랜치 작업**

메인 브랜치 유지 + Worktree로 실험적 기능 별도 진행 → 브랜치 전환 없이 동시 작업

**23. 메모리 시스템 활용**

```
"이거 기억해줘: API 키는 .env에서 관리"
```

→ 다음 대화에서도 기억하는 영구 메모리

**24. IDE 통합 (VS Code / JetBrains)**

VS Code: Claude Code 확장 설치 → 에디터에서 바로 Claude 사용

**25. 대시보드로 사용량 모니터링**

`https://console.anthropic.com/settings/usage` → 일별/주별 토큰 사용량과 비용 추적
